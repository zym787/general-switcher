# AGS 切换阀 Modbus 通信协议说明书

## 1. 协议概述

本设备作为 Modbus RTU 从站运行，支持标准 Modbus RTU 通信协议，可通过 RS232/RS485 串口与上位机通信。

| 项目 | 说明 |
|------|------|
| 通信模式 | Modbus RTU 从站 |
| 物理接口 | USART2 (RS232) + USART3 (RS485) 双通道同时收发 |
| 默认从站地址 | 1 |
| 广播地址 | 0x00 |
| 数据格式 | 8N1 (8位数据, 无校验, 1位停止位) |
| 支持功能码 | 03H (读保持寄存器), 06H (写单个保持寄存器) |
| 寄存器总数 | 100 (地址 0~99) |
| CRC校验 | Modbus CRC16 (低字节在前) |

## 2. 波特率配置

| 枚举值 | 寄存器写入值 | 实际波特率 | 帧超时时间 |
|--------|-------------|-----------|-----------|
| BAUD_9600  | 1 | 9600 bps  | 520 μs |
| BAUD_19200 | 2 | 19200 bps | 260 μs |
| BAUD_38400 | 3 | 38400 bps | 130 μs |

> 默认波特率: 9600 bps，上电从 EEPROM 读取，无效值自动回退为 9600 bps。

## 3. 功能码详解

### 3.1 功能码 03H — 读保持寄存器

**请求帧格式:**

| 字节偏移 | 内容 | 说明 |
|---------|------|------|
| 0 | 从站地址 | 1 字节 |
| 1 | 0x03 | 功能码 |
| 2 | 寄存器起始地址高字节 | 2 字节 |
| 3 | 寄存器起始地址低字节 | |
| 4 | 寄存器数量高字节 | 2 字节 |
| 5 | 寄存器数量低字节 | |
| 6 | CRC 高字节 | 2 字节 |
| 7 | CRC 低字节 | |

**响应帧格式:**

| 字节偏移 | 内容 | 说明 |
|---------|------|------|
| 0 | 从站地址 | 1 字节 |
| 1 | 0x03 | 功能码 |
| 2 | 字节数 | = 寄存器数量 × 2 |
| 3 | 数据1高字节 | 寄存器数据按地址递增 |
| 4 | 数据1低字节 | |
| ... | ... | |
| N-1 | CRC 高字节 | 2 字节 |
| N | CRC 低字节 | |

**示例 — 读地址 0010H 的 1 个寄存器:**

```
发送: 01 03 00 10 00 01 85 CF
返回: 01 03 02 12 34 B5 33
```

### 3.2 功能码 06H — 写单个保持寄存器

**请求帧格式:**

| 字节偏移 | 内容 | 说明 |
|---------|------|------|
| 0 | 从站地址 | 1 字节 |
| 1 | 0x06 | 功能码 |
| 2 | 寄存器地址高字节 | 2 字节 |
| 3 | 寄存器地址低字节 | |
| 4 | 寄存器值高字节 | 2 字节 |
| 5 | 寄存器值低字节 | |
| 6 | CRC 高字节 | 2 字节 |
| 7 | CRC 低字节 | |

**响应帧格式:** (回显请求帧)

| 字节偏移 | 内容 | 说明 |
|---------|------|------|
| 0 | 从站地址 | 1 字节 |
| 1 | 0x06 | 功能码 |
| 2 | 寄存器地址高字节 | |
| 3 | 寄存器地址低字节 | |
| 4 | 寄存器值高字节 | |
| 5 | 寄存器值低字节 | |
| 6 | CRC 高字节 | |
| 7 | CRC 低字节 | |

**示例 — 将地址 0010H 写入 674AH:**

```
发送: 01 06 00 10 67 4A 23 C8
返回: 01 06 00 10 67 4A 23 C8
```

## 4. 异常响应

当请求出错时，从站返回异常响应帧：

| 字节偏移 | 内容 | 说明 |
|---------|------|------|
| 0 | 从站地址 | |
| 1 | 功能码 | = 请求功能码 \| 0x80 |
| 2 | 异常码 | 见下表 |

**异常码定义:**

| 异常码 | 名称 | 说明 |
|-------|------|------|
| 0x01 | MB_ERROR_FUNC | 非法功能码 |
| 0x02 | MB_ERROR_ADDR | 非法寄存器地址 (超出范围) |
| 0x03 | MB_ERROR_DATA | 非法数据值 |
| 0x04 | MB_ERROR_DEVICE | 设备故障 (CRC校验失败/地址不匹配/接收溢出) |

> 广播地址 (0x00) 请求出错时不返回异常响应。

## 5. 寄存器地址映射表

| 地址 | 名称 | 读/写 | 数据类型 | 说明 |

| 1 | SET_CW | - | - | 保留 (顺时针切换，当前未启用) |
| 2 | SET_CCW | - | - | 保留 (逆时针切换，当前未启用) |
| 3 | SET_FREE | R/W | uint16 | 自由模式切换。写入值为目标通道号，同时更新电机速度 |
| 4 | REG_4 | - | - | 保留 |
| 5 | SET_ZERO | R/W | uint16 | 复位指令。写入任意值触发复位操作 |
| 6 | REG_6 | - | - | 保留 |
| 7 | REG_7 | - | - | 保留 |
| 8 | REG_8 | - | - | 保留 |

## 6. 通信时序

### 6.1 帧超时参数

| 参数 | 值 | 说明 |
|------|---|------|
| BUS_IDLE_TIME | 10 个定时周期 | 帧间空闲时间 (>3.5字符时间)，判定帧结束 |
| FRAME_ERR_TIME | 4 个定时周期 | 帧内字符间隔超时 (>1.5字符时间)，判定帧错误 |
| 定时器周期 | 45 μs (9600bps) | 由 TIM3 定时器提供，周期随波特率变化 |

### 6.2 通信流程

1. 从站默认处于接收模式
2. 收到首字节后检测是否为本机地址或广播地址
3. 地址匹配则进入接收状态，逐字节接收数据
4. 总线空闲超过 3.5 字符时间后，判定帧接收完成
5. CRC16 校验通过后，根据功能码分发处理
6. 处理完成后发送响应 (双通道: USART3 先发，USART2 后发)
7. 发送完成后切回接收模式

### 6.3 RS485 收发控制

| 控制信号 | 状态 | 说明 |
|---------|------|------|
| PB1 = 0 | 接收模式 | RX_EN() |
| PB1 = 1 | 发送模式 | TX_EN() |

## 7. 寄存器分区与访问权限总结

| 分区名称 | 地址范围 | 正常模式读 | 正常模式写 | 工厂模式读 | 工厂模式写 |
|---------|---------|----------|----------|----------|----------|
| CTRL | 0~9 | ✓ | ✓ | ✓ | ✓ |
| STATUS | 10~19 | ✓ | ✗ | ✓ | ✗ |
| OPERATE1 | 20~25 | ✓ | ✓ | ✓ | ✓ |
| OPERATE2 | 30~39 | - | - | - | - |
| USER | 40~42 | ✓ | ✓ | ✓ | ✓ |
| FACTORY1 | 50~55 | ✓ | ✗ | ✓ | ✓(DATA区) |
| FACTORY2 | 60~69 | ✓ | ✗ | ✓ | ✓ |

> 说明: 正常模式下，出厂参数区写入操作被屏蔽；工厂/老化模式下开放写入权限。

## 8. 典型操作示例

### 8.1 切换到通道 3 (就近方向)

```
发送: 01 06 00 00 00 03 48 0A    // 向地址0写入3
返回: 01 06 00 00 00 03 48 0A
```

### 8.2 自由模式切换到通道 2

```
发送: 01 06 00 03 00 02 79 A9    // 向地址3写入2，同时更新速度
返回: 01 06 00 03 00 02 79 A9
```

### 8.3 复位操作

```
发送: 01 06 00 05 00 01 98 0A    // 向地址5写入任意值触发复位
返回: 01 06 00 05 00 01 98 0A
```

### 8.4 读取当前状态 (通道+状态+耗时)

```
发送: 01 03 00 0A 00 03 61 08    // 读地址10~12
返回: 01 03 06 00 03 00 00 01 F4 xx xx  // 通道3, 状态0, 耗时500ms
```

### 8.5 进入工厂模式

```
发送: 01 06 00 09 00 FC A1 09    // 向地址9写入0xFC(252)
返回: 01 06 00 09 00 FC A1 09
```

### 8.6 设置通道数为 6 (工厂模式)

```
发送: 01 06 00 3E 00 06 99 B7    // 向地址62写入6
返回: 01 06 00 3E 00 06 99 B7
```

### 8.7 修改波特率为 19200

```
发送: 01 06 00 17 00 02 79 84    // 向地址23写入2
返回: 01 06 00 17 00 02 79 84
```

> 修改波特率后需重新上电生效。

### 8.8 读取芯片 UID

```
发送: 01 03 00 32 00 06 20 19    // 读地址50~55
返回: 01 03 0C xx xx xx xx xx xx xx xx xx xx xx xx  // 返回6个寄存器(3个UID字)
```

## 9. 注意事项

1. **双通道同时发送**: 设备响应时 USART3 (RS485) 先发送，USART2 (RS232) 后发送，两通道数据完全一致。
2. **地址 0 支持**: 支持从站地址 0 的查询请求。
3. **寄存器越界保护**: 读/写地址超出范围 (≥100) 将返回 0x02 异常码。
4. **EEPROM 持久化**: 地址、速度、波特率、通道数、半通道、序列号、补偿值等参数写入后自动保存到 EEPROM，断电不丢失。
5. **模式保护**: 出厂参数区的写入操作仅在工厂模式或老化模式下有效，正常模式下写入操作被忽略。
6. **复位指令**: 向地址 5 写入任意值触发复位，复位过程会将状态设为 VALVE_INITING (0x80)。
7. **方向固定**: 方向寄存器 (地址 22) 读取固定返回 0xFF，表示始终使用就近方向切换，当前不支持指定方向。
